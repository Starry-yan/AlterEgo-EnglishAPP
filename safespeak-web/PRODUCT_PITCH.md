## 2️⃣ 解决方案：产品核心理念

### 产品定位
> **"英语版的《模拟人生》——披着马甲的英语世界，允许犯错的乌托邦"**

### 三大核心创新

| 创新点 | 解决的问题 | 实现方式 |
|--------|-----------|---------|
| **理想自我投射** | 开口焦虑 | 创建虚拟形象，匿名交流 |
| **视觉直连** | 翻译思维 | 全英文环境，物体→英文直接映射 |
| **重述法纠错** | 挫败感 | 隐性纠正，不打断对话流 |

### 用户旅程
```
不敢开口 → 敢于开口 → 流利表达
   ↓           ↓           ↓
  AI 陪练    虚拟形象     真人社交
```

## 3️⃣ 技术实现：AI 驱动架构

### MVP 技术栈（适合单人开发）

```
┌─────────────────────────────────────────────────────┐
│                    前端 (Web)                        │
│  - HTML5 + CSS3 + Vanilla JavaScript                │
│  - Web Speech API (语音识别 + 语音合成)              │
│  - localStorage (用户数据持久化)                     │
└─────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────┐
│                   AI 能力层                          │
│  - 语音识别：浏览器内置 Web Speech API               │
│  - 对话生成：规则引擎 → GPT-4 API (后期升级)        │
│  - 语音合成：浏览器内置 SpeechSynthesis             │
└─────────────────────────────────────────────────────┘
```

### 核心算法

#### 重述法 (Recast Method)
```javascript
// 用户说："I go store yesterday"
// AI 不纠正："You should say 'went'"
// AI 重述："Oh, you went to the store? What did you buy?"
```

#### 三级递进提示系统
| 层级 | 触发时间 | 提示方式 | 设计逻辑 |
|-----|---------|---------|---------|
| L1 | 5 秒 | 语音锚点（慢速重复关键词） | 模拟真人帮助回忆 |
| L2 | 10 秒 | 首字母/高亮暗示 | 主动联想，非被动接受 |
| L3 | 15 秒 | 中文翻译 | 记录为"卡顿遗迹"用于复习 |

## 📊 专业术语解释（应对提问）

### 1. 重述法 (Recast)
> 二语习得研究中的隐性纠错策略。不直接指出错误，而是用正确的形式重述用户话语，让学习者在无压力环境下内化正确表达。

### 2. 可理解性输入 (Comprehensible Input)
> Krashen 输入假说核心概念。学习者需要接触略高于当前水平的语言材料 (i+1)，才能有效习得语言。

### 3. 情感过滤假说 (Affective Filter)
> 焦虑、缺乏自信等负面情绪会形成"心理屏障"，阻碍语言输入进入大脑的语言习得机制。Alter Ego 的虚拟形象设计正是为了降低情感过滤。

### 4. 任务型语言教学 (TBLT)
> 以完成真实任务为核心（如"点咖啡"），而非机械操练语法。Alter Ego 的场景设计遵循此原则。
